Phần 1: Phân tích logic - Nguyên nhân gây "Mất trí nhớ"
Lỗi giỏ hàng trống trơn xảy ra do sự kết hợp của cả hai yếu tố: phạm vi lưu trữ của HttpServletRequest và cơ chế hoạt động của lệnh redirect.

Bản chất của HttpServletRequest: Dữ liệu được lưu bằng request.setAttribute() có phạm vi (scope) rất ngắn. Nó chỉ tồn tại từ lúc server nhận được Request cho đến khi server trả về Response cho trình duyệt. Sau đó, Request này sẽ bị huỷ bỏ hoàn toàn.

Bản chất của redirect: Lệnh return "redirect:/checkout"; thực chất là server báo cho trình duyệt (thông qua mã HTTP 302) rằng: "Hãy tạo một HTTP GET Request mới hoàn toàn gửi tới đường dẫn /checkout đi".

Kết luận luồng chạy (Trace code):

Người dùng bấm thêm sản phẩm -> Gửi Request #1 tới /add-to-cart.

Hệ thống tạo giỏ hàng, lưu vào Request #1 và trả về lệnh redirect. Request #1 lập tức bị huỷ bỏ, mang theo cả giỏ hàng vừa tạo.

Tại hàm viewCheckout, hệ thống cố gắng tìm giỏ hàng trong Request #2. Đương nhiên là không có gì cả (null), dẫn đến hiện tượng "mất trí nhớ".
